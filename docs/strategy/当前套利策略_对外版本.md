# 当前套利策略（对外版本）

## 概述

当前 wedge 是一个基于 NOAA GEFS 集合预报的天气市场定价系统。

它的工作方式是：

1. 直接获取 NOAA GEFS ensemble 数据
2. 将某城市本地日最高温转换为概率分布
3. 将该分布与 Polymarket 天气 bucket 价格进行比较
4. 仅在预期收益为正时，使用 ladder-only 策略分步建仓
5. 在优势消失或预报显著转向时退出

## 核心思想

该系统并不依赖单一温度观点，而是依赖 **概率分布与市场定价之间的偏差**。

也就是说，它不是简单地判断“明天会不会很热”，而是判断：

> 当前市场是否低估了某个具体温度区间发生的概率。

如果市场价格低于模型给出的合理概率，并且在考虑手续费与滑点之后仍具有正期望值，系统才会执行交易。

## 数据优势

当前策略的主要优势来自以下几点：

### 1. 直接使用 NOAA GEFS

系统直接从 NOAA NOMADS 获取 GEFS 数据，而不是依赖第三方天气聚合层作为主预测源。

### 2. 使用 ensemble 分布而不是单点预报

系统不会只看一个温度值，而是会基于 ensemble members 构造完整分布，从而更适合给 bucket 市场定价。

### 3. 处理的是“本地日最高温”

市场结算通常依赖某一地点、某一自然日、某一最高温结果。系统的建模对象与市场标的本身更加一致。

## 当前策略形态

当前 runtime 已刻意收缩为：

- **Ladder-only**
- fractional Kelly sizing
- probability-based exits
- daily settlement flow

这意味着系统目前不再运行：

- tail strategy
- cross-bucket arbitrage
- telegram runtime
- online calibration in live loop

## 风险控制

当前保留的主要控制手段包括：

- 正 EV 才允许进场
- Ladder edge threshold
- fractional Kelly
- max bet / max bet percentage
- ensemble spread 折扣
- Brier threshold 暂停
- 预报反向或优势消失时提前退出

## 盈利来源

该策略的盈利来源不是“天气一定会如何”，而是：

> 市场参与者未能及时、准确地把新的概率信息反映到价格中。

在这种情况下，系统有机会通过更快、更细的概率估计，参与价格修复过程。

## 对手方画像

从市场结构上看，策略可能主要面对以下几类对手方：

- 依赖直觉或叙事交易的散户
- 使用较粗糙天气信息的参与者
- 不做动态再定价或不及时平仓的持仓者
- 对部分 bucket 报价不够精细的流动性提供者

## 边界说明

该策略并不是无风险套利，也不保证短期每一笔交易都盈利。

它本质上属于：

> 通过概率估计能力与执行纪律，持续参与错误定价修复的统计型交易系统。

## 当前版本的最准确描述

可以这样概括当前 wedge：

> 一个 direct NOAA GEFS 驱动的天气市场概率定价系统。它将 ensemble 结果转换为本地日最高温分布，将该分布与 Polymarket bucket 定价比较，并在存在正期望值时以 ladder-only 方式交易温度区间。
