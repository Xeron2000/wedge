# 当前套利策略详解

## 一句话定义

当前的 wedge 不是无风险套利系统，而是一个 **基于 NOAA/GEFS 概率分布的天气定价偏差交易系统**。

更准确地说，它做的是：

> 使用 direct NOAA GEFS ensemble data 估计某城市本地日最高温的概率分布，将这个分布与 Polymarket 天气 bucket 的市场价格比较；如果扣除手续费和滑点后仍然存在正期望值，就用 ladder-only 策略建仓，并在预报转向或优势消失时退出。

## 当前策略的完整结构

### 1. 数据源：direct NOAA GEFS

系统现在直接从 NOAA NOMADS 获取 GEFS ensemble 数据，而不是把 Open-Meteo 当预测主源。

实际处理流程是：

1. 选择最新的 GEFS cycle（00/06/12/18 UTC）
2. 针对目标城市机场坐标抓取 member GRIB 数据
3. 提取 `2m above ground` 的温度预报
4. 将目标自然日内多个 3 小时 forecast step 聚合成 **每个 member 的本地日最高温**

最终得到的不是一个单点温度，而是多个 member 对“这一天最高温”的独立看法。

### 2. 概率分布构造

系统会把所有 member 的目标日最高温结果桶化，形成温度概率分布。

例如：

- 80°F：12%
- 81°F：18%
- 82°F：26%
- 83°F：21%
- 84°F：13%
- 85°F：10%

这个分布就是模型对未来结算结果的 `p_model` 表达。

这一步是策略的核心，因为 Polymarket 交易的正是温度 bucket 发生概率，而不是一句模糊的“明天会热”。

### 3. 市场扫描

系统会扫描 Polymarket 对应城市、对应日期的天气市场，提取：

- bucket 温度值
- 市场价格 `p_market`
- token_id
- 温度单位

在这个策略里，市场价格就等价于市场隐含概率。

### 4. edge 与 EV 计算

系统先计算：

- raw edge = `p_model - p_market`

但真正用于决策的不是 raw edge，而是 EV：

\[
EV = p_{model} \times (1-fee) \times odds - (1-p_{model}) - slippage
\]

其中：

- `odds = (1 - p_market) / p_market`
- `fee_rate` 默认 2%
- `slippage` 根据市场成交量估算

因此，系统只会在以下条件同时满足时出手：

1. 模型概率高于市场概率
2. 扣完手续费和滑点后，仍然是正期望值
3. edge 超过 ladder 阈值

### 5. 只做 Ladder，不做 Tail

当前 runtime 已经收缩成 **Ladder-only**。

这意味着系统不再运行：

- tail 极端天气策略
- cross-bucket arbitrage
- 事件驱动交易
- telegram runtime
- 在线 calibration runtime

当前只做一件事：

> 在市场明显低估的温度区间上，按 edge 从高到低分层建仓。

默认参数：

- `ladder_edge = 0.08`
- `ladder_alloc = 0.90`

也就是至少需要约 8% 的模型-市场概率偏差，才会进入 ladder 候选集。

### 6. 仓位控制：fractional Kelly

策略不会因为发现 edge 就满仓，而是通过 fractional Kelly 控制仓位。

当前控制因素包括：

- `kelly_fraction = 0.15`
- `max_bet`
- `max_bet_pct`
- `ensemble_spread` 折扣
- `fat-tail discount`

也就是说，模型虽然可能看多某个 bucket，但如果预报分布太散、市场太差或仓位上限已接近，最终下单金额仍会被压缩。

### 7. 退出策略

当前退出逻辑有三层：

#### 止损
如果最新 `p_model` 下降到：

\[
p_{model} < entry\_price \times exit\_loss\_factor
\]

默认 `exit_loss_factor = 0.5`。

含义是：模型已经明显不再支持原先入场逻辑，提前认错离场。

#### 优势消失退出
如果：

- `p_model >= entry_price`
- 但 `EV <= 0`

说明方向未必完全错，但价格已经被市场修复，edge 不再存在。

这时提前退出，锁定已经实现的 repricing。

#### 临近结算不动
如果距离结算太近，系统会避免提前退出，以免被最后阶段噪音扰动。

### 8. 结算

结算阶段仍使用 archive observed temperatures 获取历史日最高温，以完成 settled PnL 的计算。

## 这个策略为什么可能赚钱

### 1. 它赚的是“定价修复的钱”，不是“天气一定会发生的钱”

这个系统不是在赌某个 bucket 一定会中，而是在赌：

> 市场当前给这个 bucket 的赔率太便宜。

例如：

- 市场价格 0.20
- 模型概率 0.32

那么策略下注的不是“83°F 一定发生”，而是“0.20 这个定价长期来看太低”。

### 2. 你比市场更快把 NOAA 原始 ensemble 变成可交易概率

很多市场参与者并不会做以下几步：

- 直接抓 NOAA GEFS
- 按机场点位取值
- 聚合成本地日最高温
- 做 member 分布
- 转成 bucket probability
- 再做 fee/slippage 后的 EV 判断

因此 wedge 的 edge 不是一句“我更懂天气”，而是：

> 我能更快、更细、更系统地把原始 forecast 变成交易概率。

### 3. 市场常常只交易方向，不交易分布

天气市场最常见的错误之一，是大家只看：

- 会不会热
- 会不会冷
- 会不会破纪录

但真正交易的是：

> 某个具体温度 bucket 是否发生。

bucket 定价需要处理很多细节：

- 是日最高温，不是瞬时温度
- 是本地自然日，不是 UTC 日
- 是机场点位，不是市中心
- 是分布，不是单点 forecast

市场参与者如果没有认真处理这些细节，就会产生错误定价。

### 4. 价格修复并不总是瞬时完成

即使 NOAA 新 cycle 已经出来，市场也不一定立刻修完价，原因包括：

- 很多人根本不看 NOAA 原始数据
- 做市商不一定逐桶精细重定价
- 散户订单流会把价格推向情绪方向
- 流动性薄的 bucket 修复更慢
- 临近天气事件时，叙事会先走，数据会后纠偏

因此 wedge 赚的是：

> 市场还没有把新概率完全反映进价格之前的窗口。

## 赚谁的钱

严格说，赚的是 **错误定价对手盘** 的钱。主要包括以下几类。

### 1. 凭感觉下注的散户

他们更像在交易 narrative：

- 今天感觉很热
- 新闻说要降温
- 大家都在说极端天气
- 这个市场已经涨了还会涨

而 wedge 在交易的是 bucket probability。

所以一旦这类资金把价格推偏，你就在赚他们的错误定价。

### 2. 只看二手天气数据的人

这类人可能看：

- 天气 app
- 单点 forecast
- 聚合 forecast
- 不是 direct NOAA 的包装源

他们也许不是完全外行，但没有把原始 ensemble 做成你现在这种可交易分布，因此定价通常更粗。

### 3. 不会动态再定价或不及时平仓的人

很多人买入后不会根据 forecast 更新而及时调整仓位。

表现为：

- 预报已经转向，但还抱着旧仓位
- 市场已经修复，但还不退出
- 持仓逻辑是“等结算”，而不是“等 edge 消失就走”

你的退出策略就是在赚这类慢钱。

### 4. 报价粗糙的流动性提供者

并不是所有做市资金都会认真用 NOAA ensemble 给每个 bucket 精确定价。

很多时候它们只是：

- 粗粒度报价
- 半自动调价
- 对热门 bucket 重视，对冷门 bucket 粗放

这时 wedge 会吃到这些“报价不够细”的钱。

## 这个策略不是什么

当前策略不是：

- 无风险套利
- 必赚系统
- 只靠方向判断的主观下注
- 结构性 bucket sum arbitrage
- 事件驱动预测系统

它本质上是：

> 一个 NOAA/GEFS 驱动的正期望值天气赔率交易系统。

## 当前最准确的策略表述

可以这样定义当前 wedge：

> Wedge 是一个 direct NOAA GEFS 驱动的天气市场定价机器人。它把 ensemble member 转换为某城市本地日最高温的概率分布，将该分布与 Polymarket 的温度 bucket 价格比较，只在扣除手续费和滑点后仍存在正期望值时，用 Ladder-only + fractional Kelly 建仓，并在预报转向或优势消失时退出。
