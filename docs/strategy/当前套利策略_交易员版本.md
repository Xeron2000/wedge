# 当前套利策略（交易员版本）

## 策略核心

当前 wedge 的交易逻辑可以压缩成一句话：

> 用 direct NOAA GEFS 做本地日最高温概率分布，抓 Polymarket bucket 的错误定价，只做 ladder，赚市场对新概率反应不够快的钱。

## 交易链路

### 1. 预测侧

- 直接抓 NOAA NOMADS GEFS
- 以机场对齐坐标提取 2m temperature
- 把目标日多个 3 小时 step 聚合成 member 的 daily max
- 用 member daily max 构成 bucket probability distribution

最终产物不是“明天 83 度”，而是：

- 80°F：x%
- 81°F：y%
- 82°F：z%

### 2. 市场侧

- 扫对应 city/date 的 Polymarket weather buckets
- 读取 `market_price`
- 视作 bucket 隐含概率 `p_market`

### 3. 定价逻辑

- raw edge = `p_model - p_market`
- 真正决策变量是 EV，而不是裸 edge

公式：

\[
EV = p_{model} \times (1-fee) \times odds - (1-p_{model}) - slippage
\]

含义：

- 市场赔率再便宜，如果扣掉 fee / slippage 之后不赚钱，也不碰
- 只有真正正 EV 的 bucket 才进候选池

## 为什么只做 Ladder

当前 runtime 已经主动放弃：

- tail
- arbitrage
- events
- telegram runtime
- calibration runtime

原因很简单：

- 先把最稳定、最容易解释、最接近主 thesis 的部分跑通
- 中心温度区间通常流动性更好、噪音更小、样本更多
- tail 很容易把系统从“赔率交易”拖成“观点赌博”

所以现在只做：

> 最有把握、最能验证 edge 的那一段价格偏差。

## 下单逻辑

### 过滤

必须同时满足：

- `edge > ladder_edge`
- `EV > 0`

默认阈值：

- `ladder_edge = 0.08`

### 排序

- 按 edge 从高到低
- 优先吃最便宜、最错价的 bucket

### 仓位

用 fractional Kelly：

- `kelly_fraction = 0.15`
- `max_bet`
- `max_bet_pct`
- `ensemble_spread` 折扣
- `fat-tail discount`

本质是：

> 有 edge 才下注；有 edge 也不重仓；预报越散，仓位越小。

## 退出逻辑

### 止损

如果最新 `p_model` 塌到：

\[
p_{model} < entry\_price \times exit\_loss\_factor
\]

默认 `exit_loss_factor = 0.5`

含义：模型已经不支持原来的交易理由，直接撤。

### 优势消失退出

如果：

- `p_model >= entry_price`
- `EV <= 0`

说明方向还没完全反过来，但市场已经修复到没肉了，离场。

### 临近结算不动

- 太靠近结算就不再频繁折腾
- 避免最后阶段噪音把本来正确的仓位洗掉

## 赚谁的钱

### 1. 情绪散户

- 看感觉
- 看新闻
- 看 narrative
- 不看 probability distribution

### 2. 二手天气数据玩家

- 看 app forecast
- 看单点 forecast
- 不看 raw ensemble
- 不做 member → daily max → distribution 这条转换

### 3. 持有到结算的慢钱

- forecast 已经变了不走
- edge 已经没了不走
- 不做动态再定价

### 4. 粗糙做市钱

- 不会逐 bucket 精细报价
- 对部分 bucket 反应慢
- 给出“方向对、价格错”的盘口

## 为什么它可能赚钱

交易员视角最简洁的回答就是：

> 市场在交易天气故事，你在交易天气分布。

更细一点：

- 别人看的是热不热、冷不冷
- 你看的是每个 bucket 的条件概率
- 别人用 narrative 下单
- 你用 EV 下单
- 别人很多时候只会开仓
- 你在管理进场、再定价和退出

## 策略边界

这不是：

- 无风险套利
- bucket sum arbitrage
- 极端天气尾部彩票系统
- 纯主观天气观点下注

它是：

> 一个靠 NOAA/GEFS 概率处理能力吃错误定价修复的钱的系统。

## 交易员版结论

如果一句话给交易员解释当前 wedge：

> wedge 现在做的是 NOAA GEFS repricing trade：把原始 ensemble 快速翻译成 bucket probability，专门买市场还没来得及修价、且扣完 fee/slippage 后仍然为正 EV 的温度 bucket，并用 ladder + Kelly 收割慢反应对手盘。
