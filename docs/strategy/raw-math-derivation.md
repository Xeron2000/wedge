  ---
  策略核心数学推导

  基本定理：天气阶梯套利

  买入 N 个相邻温度档位，这本质上是一个区间二元期权：

  总成本 = Σ c_i  (各档位市场价之和)
  命中任一档位：净利 = $1 - 总成本
  全部未命中：  净亏 = -总成本

  EV = P_hit × $1 - 总成本

  只要 P_hit > 总成本，就是正EV。 极其简洁。

  具体计算：NYC 3日后高温

  GFS 31成员集合预报分布：

  ┌──────────┬────────┬─────────┬────────┬──────────┐
  │ 温度(°F) │ 成员数 │ P_model │ 市场价 │ P_market │
  ├──────────┼────────┼─────────┼────────┼──────────┤
  │ ≤76      │ 3      │ 9.7%    │ $0.11  │ 11.0%    │
  ├──────────┼────────┼─────────┼────────┼──────────┤
  │ 77       │ 4      │ 12.9%   │ $0.10  │ 10.0%    │
  ├──────────┼────────┼─────────┼────────┼──────────┤
  │ 78       │ 7      │ 22.6%   │ $0.18  │ 18.0%    │
  ├──────────┼────────┼─────────┼────────┼──────────┤
  │ 79       │ 8      │ 25.8%   │ $0.22  │ 22.0%    │
  ├──────────┼────────┼─────────┼────────┼──────────┤
  │ 80       │ 5      │ 16.1%   │ $0.18  │ 18.0%    │
  ├──────────┼────────┼─────────┼────────┼──────────┤
  │ 81       │ 3      │ 9.7%    │ $0.10  │ 10.0%    │
  ├──────────┼────────┼─────────┼────────┼──────────┤
  │ ≥82      │ 1      │ 3.2%    │ $0.06  │ 6.0%     │
  └──────────┴────────┴─────────┴────────┴──────────┘

  阶梯操作：买入 77-80°F 区间

  总成本     = $0.10 + $0.18 + $0.22 + $0.18 = $0.68
  P_model    = 12.9% + 22.6% + 25.8% + 16.1% = 77.4%
  P_market   = 10% + 18% + 22% + 18%          = 68.0%
  Edge       = 77.4% - 68.0%                  = 9.4%
  EV/每组    = 0.774 × $1 - $0.68             = +$0.094

  9.4%的edge，在任何量化交易中都算丰厚。

  Kelly仓位计算

  b（赔率） = ($1 - $0.68) / $0.68 = 0.471
  f*（满Kelly） = (0.774 × 0.471 - 0.226) / 0.471 = 29.4%
  f_fractional（15%分数Kelly） = 29.4% × 0.15 = 4.4% bankroll

  Thorp 的原则：永远用分数Kelly，满Kelly的几何增长是理论值，实际波动会杀死你。 文档中的15%分数Kelly是保守且正确的。

  $1000本金 → 每组下注$44 → 可买64组(64 × $0.68)

  命中(77.4%概率)：利润 = 64 × $0.32 = +$20.48
  未命中(22.6%)：  亏损 = -$44.00
  E[单次] = 0.774 × $20.48 - 0.226 × $44 = +$5.91

  ---
  三个Edge来源的量化分析

  Edge 1：模型更新时间差（最可靠）

  GFS更新周期：00Z/06Z/12Z/18Z，数据延迟~4小时
  Edge半衰期：

    0-30 min 后获取数据  → 100% edge
    30-60 min            → ~70%
    60-120 min           → ~40%
    120+ min             → <20%（被其他bot抹平）

  操作窗口：数据发布后30分钟内执行。 这是最硬的edge——纯信息速度优势。每天4次GFS更新 = 4个交易窗口。

  Edge 2：散户认知偏差（最稳定）

  散户典型错误：
  - 锚定效应：用昨天的温度预测明天 → 系统性滞后
  - 忽视ensemble spread：只看点预测，不看概率分布
  - 近因偏差：连续几天30°C就认为明天还是30°C

  这个edge不会被bot消除，因为散户持续进入市场。Silver的话：

  ▎ "Most people confuse a forecast with a prediction. A forecast is a probability distribution."

  Edge 3：做市商定价偏差（最大但最不稳定）

  AMM/订单簿结构性问题：
  - 各档位价格之和经常 ≠ 100%（存在vigorish或定价gap）
  - 尾部档位被系统性高估或低估
  - 做市商用简单模型（正态分布），而天气分布经常是偏态的

  ---
  双层策略组合

  Thorp的杠铃，不是Taleb的杠铃：

  层1：区间阶梯（70%资金，低波动）

  目标：每日稳定正EV
  操作：买入模型预测概率中心区域(±1σ)的3-5个档位
  触发：range edge > 5%
  Kelly：15% fractional
  每月~30次交易
  E[月收益]：30 × $5.91 = $177（17.7%月回报）

  层2：尾部猎手（20%资金，高波动）

  目标：捕获极端天气事件的巨额赔率
  操作：当ensemble中≥3个成员预测极端温度，
        而市场定价仅2-5%时买入

  示例：
    市场价：95°F+ = $0.03（3%隐含概率）
    模型：4/31成员预测95°F+ = 12.9%
    Edge = 9.9%
    赔率 = 32.3:1
    Kelly = 1.5% bankroll

    $1000 × 1.5% = $15下注
    命中：$15/$0.03 × $1 = +$485
    未命中：-$15
    EV = 0.129 × $485 - 0.871 × $15 = +$49.50

  层3：现金储备（10%）

  应对drawdown和突发机会。

  年化收益预估

  月收益构成：
    阶梯：~$177（稳定）
    尾部：~5次 × $49.50 × 12.9%命中率 = ~$32（高方差）
    合计：~$209/月

  年化（含复利，$1000起）：
    Month 1:  $1,000 → $1,209
    Month 3:  ~$1,768
    Month 6:  ~$3,126
    Month 12: ~$9,771

    年化ROI: ~877%（理论值）

  Thorp的警告： 理论值打5-7折才是现实。考虑：
  - 流动性摩擦（大单滑点）→ -20%
  - 模型误差（GFS不是真理）→ -15%
  - 竞争加剧（其他bot）→ -10%
  - 执行延迟 → -5%

  现实预估：年化300-400%，$1000 → $3000-5000。 对于$1000级别资金已经非常优秀。

  ---
  关键约束

  1. 资金天花板：天气市场流动性有限，$5000-10000以上就开始有显著滑点
  2. 模型降级风险：GFS偶尔会出现系统性偏差，需要交叉验证ECMWF
  3. Brier Score监控：持续追踪你的预测准确度，如果Brier Score > 0.25就暂停交易
  4. 不要用满Kelly：文档中的15%分数Kelly是底线，我建议10-15%
